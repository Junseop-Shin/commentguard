# 로컬 테스트 가이드

## 1. 빌드

```bash
npm run build
```

`dist/` 폴더가 생성되면 준비 완료.

---

## 2. Chrome에 로드

1. Chrome 주소창에 `chrome://extensions` 입력
2. 우상단 **개발자 모드** 토글 ON
3. **"압축해제된 확장 프로그램을 로드합니다"** 클릭
4. `dist/` 폴더 선택

> 설정 변경 시: `npm run build` 재실행 → 확장 프로그램 카드의 새로고침(↺) 버튼 클릭

---

## 3. 기능 점검 체크리스트

### 기본 필터링

- [ ] YouTube 영상 댓글 섹션 진입 시 봇 댓글이 블라인드 처리됨
- [ ] 블라인드 댓글이 "🚫 필터링 된 댓글입니다 [펼쳐보기]" 플레이스홀더로 표시됨
- [ ] **[펼쳐보기]** 버튼 클릭 시 원본 댓글이 복원됨
- [ ] 펼친 댓글은 페이지 스크롤/재스캔 후에도 다시 블라인드되지 않음

### 키워드 필터

- [ ] 팝업 → 키워드 추가 후 해당 단어가 포함된 댓글이 블라인드됨
- [ ] 키워드 토글(비활성화) 시 해당 댓글이 즉시 복원됨
- [ ] 키워드 삭제 시 해당 댓글이 즉시 복원됨
- [ ] 초성 키워드 (예: `ㅂㅋ`) 가 일치하는 댓글을 필터링함

### 닉네임 필터

- [ ] 팝업 → 닉네임 추가 후 해당 닉네임 작성자의 댓글이 블라인드됨
- [ ] 닉네임 삭제 시 해당 댓글 즉시 복원됨

### 프리셋 (예: 홍보, 도박)

- [ ] 프리셋 카테고리 ON/OFF 토글이 즉시 반영됨

### 봇 감지

- [ ] 봇 패턴 댓글(URL + 홍보 문구 조합 등) 이 블라인드됨
- [ ] 민감도 슬라이더 조절 후 적용됨 (낮추면 덜 필터링)

### 고정 댓글 보호

- [ ] "고정 댓글 보호" ON 상태에서 핀 배지 댓글은 필터링되지 않음
- [ ] OFF 상태에서는 고정 댓글도 키워드에 걸리면 필터링됨

### 한국어 댓글 우선 정렬

- [ ] 팝업 → "한국어 댓글 우선 정렬" ON 시 한글 댓글이 목록 상단으로 이동
- [ ] OFF 시 원래 순서로 복원됨
- [ ] 정렬 중 새 댓글 로드(스크롤) 시에도 정렬 순서 유지됨

### 다크 모드

- [ ] YouTube 다크 테마 ON 시 플레이스홀더 배경/텍스트 색상이 다크 스타일로 표시됨
- [ ] OS 다크 모드에서도 동일하게 적용됨

### 통계

- [ ] 팝업 → 통계 탭에서 필터링 건수가 정확히 집계됨
- [ ] 통계 초기화 버튼이 정상 동작함

### 페이지 이동

- [ ] 다른 YouTube 영상으로 이동 후에도 필터링이 정상 작동함
- [ ] 영상 → 홈 → 영상 반복 이동 시 메모리 누수나 오류 없음 (콘솔 확인)

---

## 4. 콘솔 오류 확인

1. YouTube 댓글 있는 영상 진입
2. F12 → Console 탭
3. `[YT Filter]` 접두사 오류 없는지 확인
4. MutationObserver 관련 경고 없는지 확인

---

## 5. 스토리지 확인

```
F12 → Application → Storage → chrome.storage.local
```

- `yt-comment-filter`: 설정/키워드/닉네임/프리셋
- `yt-cf-stats`: 필터링 통계 (별도 키로 분리됨)
