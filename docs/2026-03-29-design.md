# YT Comment Filter — Design Spec
> Created: 2026-03-29

## Overview

YouTube 댓글 필터링 Chrome Extension.
키워드/닉네임/봇 패턴 차단 + 한글 특화 + 로컬 통계. 서버 없음, 무료 배포.

---

## Tech Stack

| 항목 | 선택 |
|------|------|
| 빌드 | Vite + TypeScript |
| UI | Vue 3 + Pinia (팝업) |
| 스타일 | Tailwind CSS |
| 한글 처리 | es-hangul (toss) |
| 저장소 | chrome.storage.local |
| 배포 | Chrome Web Store (무료) |

## Data Model

```ts
interface FilterStore {
  enabled: boolean;

  filters: {
    keywords:    FilterItem[];
    nicknames:   FilterItem[];
    botPatterns: BotRule[];
  };

  presets: {
    [category: string]: {
      enabled:  boolean;
      keywords: string[];
    };
  };

  settings: {
    useChosung:          boolean;  // 초성 검색 on/off
    useNormalize:        boolean;  // 변형문자 정규화 on/off
    botSensitivity:      number;   // 0~100 임계값 (기본 70)
    protectPinnedComment: boolean; // 고정 댓글 차단 제외 (기본 true)
    showBlockedToggle:   boolean;  // 차단된 댓글 임시 보기 허용
  };

  stats: {
    total:     number;
    byKeyword: Record<string, number>;  // 키워드별 차단 횟수
    since:     number;                  // 통계 시작 timestamp
  };
}

interface FilterItem {
  id:        string;
  value:     string;
  enabled:   boolean;
  isPreset:  boolean;
}
```

    keywords: ["도박", "바카라", "카지노", "입금보너스", "텔레그램", "토토"],

  "자기홍보": {
    enabled: true,
    keywords: ["구독", "좋아요", "홍보", "check my", "내 채널"],
  },
  "성인": {
    enabled: true,

## Filtering Pipeline

```
댓글 DOM 감지 (MutationObserver + requestIdleCallback)
        ↓
  고정 댓글 여부 확인 (#pinned-comment-badge)
  → 고정 댓글이면 스킵 (protectPinnedComment: true 시)
        ↓
  텍스트 / 닉네임 추출
  (ytd-comment-view-model)
        ↓
  [normalizer.ts]
  1. String.normalize('NFKC')     → 전각 문자 → 반각
  2. 치환맵 적용                  → 1→ㅣ, |→ㅣ, 0→ㅇ 등
  3. 특수문자 제거 후 비교용 복사본 생성
        ↓
  [filter.ts]
  1. 키워드 매칭
     - 일반: includes()
     - 초성: es-hangul chosungIncludes()
     - 정규화 텍스트 대상 추가 비교
  2. 닉네임 매칭 (like: includes())
  3. 봇 점수 계산 → 임계값 초과 시 차단
        ↓
  차단 결정 → element.style.display = 'none'
           → stats 업데이트 (chrome.storage.local)
```

## Bot Detection (점수제, 0~100)

| 규칙 | 점수 |
|------|------|
| URL / 링크 포함 | +40 |
| 홍보 키워드 다수 (구독, 수익, 카톡 등) | +20 |
| 느낌표 2개 이상 연속 (`!!`, `！！`) | +10 |
| 특수문자 밀도 > 30% | +10 |
| 동일 단어 3회 이상 반복 | +10 |
| 한영 혼용 비율 비정상 (영문 > 60%) | +10 |

→ `botSensitivity` 슬라이더로 임계값 조절 (기본: 70점)
→ 운영 후 피드백 기반으로 점수 튜닝 예정

## Normalizer — 치환맵 (Best Effort)

단순 치환만 지원. 창의적 조합형 변형은 커버 불가 (known limitation).

```ts
const SUBSTITUTION_MAP: Record<string, string> = {
  '1': 'ㅣ', '|': 'ㅣ', 'l': 'ㅣ',
  '0': 'ㅇ',
  '$': 'ㅅ',
  '@': 'ㅇ',
};
```

## Known Limitations

- 창의적 조합형 변형 한글 (ㅆ1발 + 기타 조합) 완벽 감지 불가 → Best Effort 치환맵만 제공
- YouTube DOM 구조 변경 시 셀렉터 업데이트 필요
- 봇 감지는 휴리스틱 기반 → 과필터링/미필터링 일부 발생 가능
